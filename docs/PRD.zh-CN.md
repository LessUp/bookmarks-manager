## 产品概览

### 愿景

Bookmarks Manager 是一款**隐私优先、本地优先**的书签管理工具。所有处理都在浏览器中完成——数据永远不会离开用户设备。

### 核心原则

| 原则 | 说明 | 优先级 |
|-----------|-------------|----------|
| **隐私优先** | 零数据传输到服务器 | P0 |
| **本地优先** | 离线工作，无云端依赖 | P0 |
| **BYOK** | AI 功能自备密钥 | P1 |
| **渐进增强** | 核心功能无需 AI | P0 |

### 价值主张

**普通用户**: 在完全私密的环境下合并、去重和分析多个浏览器的书签。

**高级用户**: 使用自己的 API 密钥获得 AI 驱动的洞察。

## 目标用户

### 主要用户画像

#### 1. 多浏览器用户
- 工作用 Chrome，个人用 Firefox，手机用 Safari
- 在多个浏览器中有数千个书签
- 希望整合而不丢失组织

#### 2. 隐私意识强的整理者
- 不愿使用云端书签管理器
- 重视完全数据控制
- 偏好离线工具

#### 3. AI 辅助的高级用户
- 熟悉 LLM API
- 需要智能分类和洞察
- 愿意配置自己的 API 密钥

### 用户特征

| 特征 | 重要性 |
|----------------|------------|
| 技术素养 | 中到高 |
| 隐私意识 | 高 |
| 书签数量 | 100 - 10,000+ |
| 浏览器多样性 | 2+ 浏览器 |

## 用户故事

### 导入与合并

**US-001**: 作为用户，我希望从多个浏览器导入书签，以便将它们整合在一个地方。

**US-002**: 作为用户，我希望看到导入文件的预览，以便验证导入是否成功。

**US-003**: 作为用户，我希望重复书签被自动检测，以便不必手动查找。

**US-004**: 作为用户，我希望导入后能够移除特定的源文件，以便纠正错误。

### 搜索与发现

**US-005**: 作为用户，我希望按标题、URL 或文件夹搜索书签，以便快速找到所需内容。

**US-006**: 作为用户，我希望搜索结果高亮匹配词，以便看到匹配原因。

**US-007**: 作为用户，我希望按域名、日期或文件夹过滤书签，以便缩小结果范围。

**US-008**: 作为用户，我希望模糊搜索，以便轻微拼写错误也能找到书签。

### 分析与洞察

**US-009**: 作为用户，我希望看到书签统计，以便更好地了解我的收藏。

**US-010**: 作为用户，我希望书签趋势可视化，以便发现模式。

**US-011**: 作为高级用户，我希望 AI 为书签分类，以便高效重新组织。

**US-012**: 作为高级用户，我希望 AI 生成书签页面的摘要，以便快速回忆内容。

### 导出与备份

**US-013**: 作为用户，我希望将书签导出回浏览器，以便恢复整理后的收藏。

**US-014**: 作为用户，我希望多种导出格式，以便在不同场景使用书签。

**US-015**: 作为用户，我希望备份所有应用数据，以便迁移到另一设备。

## 功能需求

### P0 - 核心功能（必须有）

#### F-001: 多浏览器导入
- 支持 Netscape Bookmark HTML 格式
- 处理 Chrome、Firefox、Edge、Safari 导出
- 拖拽文件上传
- 大文件进度指示

#### F-002: 去重引擎
- URL 规范化（大小写、协议、参数）
- 智能重复检测
- 保留最早添加的书签
- 显示重复簇供查看

#### F-003: 全文搜索
- 索引标题、URL 和路径
- 即时结果（1万条书签 < 100ms）
- 结果高亮
- 模糊匹配支持

#### F-004: 统计仪表盘
- 书签总数
- 重复数量和比例
- 域名分布图表
- 年度趋势可视化

#### F-005: 导出功能
- Netscape HTML 格式（浏览器兼容）
- 保留文件夹结构选项
- 平铺导出选项

#### F-006: 数据持久化
- 自动保存到 IndexedDB
- 页面刷新后数据保留
- 应用启动时加载

### P1 - 增强功能（应该有）

#### F-101: 高级导出格式
- 带元数据的 JSON
- 用于表格分析的 CSV
- 用于文档的 Markdown

#### F-102: 备份与恢复
- 完整应用数据导出
- 选择性备份（书签、AI 配置等）
- 带预览的备份导入

#### F-103: 性能优化
- 大数据集 Web Worker（>500 书签）
- 搜索结果虚拟滚动（>200 条）
- 搜索输入防抖

#### F-104: AI 集成（BYOK）
- OpenAI、Claude 和自定义端点支持
- 书签分类
- 摘要生成
- 自然语言搜索
- 用量追踪和限额

### P2 - 最好有

#### F-201: 批量编辑
- 多选书签
- 批量标签分配
- 批量删除

#### F-202: 高级过滤 UI
- 日期范围选择器
- 多域名选择
- 保存的过滤器

#### F-203: 插件系统
- 自定义导出器插件
- 第三方集成钩子

## 验收标准

### AC-001: 导入流程

**假如** 用户从 Chrome 导出了书签  
**当** 他们将 HTML 文件拖到上传区  
**那么** 书签被导入并显示在文件列表中  
**并且** 统计更新以反映导入数量

### AC-002: 去重

**假如** 用户导入了包含重复的书签  
**当** 他们点击"合并去重"  
**那么** 重复被检测并移除  
**并且** 保留最早的书签  
**并且** 统计显示重复数量

### AC-003: 搜索

**假如** 书签已加载  
**当** 用户在搜索框输入"react"  
**那么** 结果在 100ms 内更新  
**并且** 结果包含标题或 URL 中有"react"的书签  
**并且** 匹配词高亮显示

### AC-004: AI 分类

**假如** 用户已配置 OpenAI API 密钥  
**当** 用户请求 AI 分类  
**那么** API 被调用并传入书签  
**并且** 结果显示分类  
**并且** 用量被记录

### AC-005: 导出

**假如** 用户已合并书签  
**当** 用户点击 HTML 导出  
**那么** 生成有效的 Netscape Bookmark HTML  
**并且** 文件下载，文件名包含时间戳
