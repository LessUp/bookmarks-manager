# 快速入门指南

## 5分钟上手 Bookmarks Analysis

### 第一步：导出浏览器书签

#### Chrome / Edge / Brave
1. 打开浏览器，按 `Ctrl+Shift+O` (Windows) 或 `Cmd+Shift+O` (Mac) 打开书签管理器
2. 点击右上角的 `⋮` 菜单
3. 选择"导出书签"
4. 保存为 HTML 文件

#### Firefox
1. 按 `Ctrl+Shift+B` (Windows) 或 `Cmd+Shift+B` (Mac) 打开书签库
2. 点击"导入和备份"
3. 选择"导出书签到 HTML"
4. 保存文件

#### Safari
1. 打开 Safari，选择"文件" > "导出书签"
2. 保存为 HTML 文件

### 第二步：启动应用

```bash
# 克隆或下载项目
cd bookmarks-analysis

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问 http://localhost:5173/

### 第三步：上传并合并

1. 在"上传合并"页面，点击或拖拽上传你导出的书签文件
2. 可以一次上传多个文件（比如 Chrome 和 Firefox 的书签）
3. 点击"合并去重"按钮
4. 等待处理完成，查看统计结果

### 第四步：探索功能

#### 📊 查看仪表盘
- 切换到"仪表盘"页面
- 查看重复占比、域名分布、年份趋势
- 点击"书签列表"查看所有书签

#### 🔍 搜索书签
- 切换到"搜索"页面
- 输入关键词（支持标题、URL、文件夹名）
- 实时查看搜索结果

#### 🔄 管理重复
- 切换到"去重"页面
- 查看所有重复的书签簇
- 了解哪些被保留、哪些被标记为重复

#### 📤 导出结果
- 返回"上传合并"页面
- 点击"导出 HTML"
- 获得一个合并去重后的书签文件
- 可以导入回任意浏览器

### 第五步：数据持久化

**好消息**：你的数据已经自动保存！

- 合并去重后，数据自动保存到浏览器的 IndexedDB
- 关闭页面或刷新后，数据依然存在
- 下次打开应用，自动加载上次的数据

如需清空数据：
- 在"上传合并"页面点击"清空"按钮
- 或在浏览器中清除站点数据

## 常见问题

### Q: 我的数据安全吗？
A: 完全安全！所有数据处理都在你的浏览器本地完成，不会上传到任何服务器。

### Q: 支持多大的书签文件？
A: 理论上没有限制，但建议单次处理不超过 10000 条书签以获得最佳性能。

### Q: 如何备份我的数据？
A: 使用"导出 HTML"功能导出书签文件，这就是你的备份。

### Q: 去重的规则是什么？
A: 系统会规范化 URL（统一大小写、排序参数、移除追踪参数），然后识别相同的链接，保留添加时间最早的那个。

### Q: 可以在手机上使用吗？
A: 可以！这是一个 PWA 应用，支持手机浏览器，也可以添加到主屏幕使用。

### Q: 数据会过期吗？
A: 不会。数据存储在浏览器的 IndexedDB 中，除非你手动清除，否则会一直保留。
